# Regression Case

Structure for a reusable test case that checks whether the skill can run the
full Level 0→4 workflow and reach a sound, evidence-grounded decision. Use real
anonymized projects when possible; use a constructed case (like the golden case)
when real data cannot be shared.

A good regression case fixes the **inputs** and an **ideal output direction**, so
re-running the skill after edits shows whether quality held, improved, or
regressed.

## Acceptance Criteria (must-haves)

- [ ] Separates fact (Level 1) from inference (Level 2) explicitly
- [ ] Every major claim traces to an evidence item
- [ ] Inference strength labeled (high/medium/low/speculative)
- [ ] Cultural tension uses What is / What should be / Why now
- [ ] Idea Platform = tension × brand truth × proof edge (not a slogan)
- [ ] Six-check applied to candidates
- [ ] Recommendation status matches input readiness honestly

## Known Traps (a weak run does this)

- Treats high-frequency words as insight on their own
- Invents brand truth not in the package to force a Final call
- Skips the provisional/rollback call when brand facts are thin
- Collapses divergent thinking too early (no real Level 1–3 expansion)
