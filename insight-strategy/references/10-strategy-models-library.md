# Strategy Models Library

Use this optional library when Level 4 needs more than the default Idea
Platform route, or when the user asks to compare strategy models. This file is
a model-selection aid, not the governing workflow.

Default governing logic remains:

```text
Idea Platform = Cultural Tension x Brand Truth x Proof Edge
```

Use other models to generate contrast candidates, sharpen positioning, or check
whether the default platform has missed a stronger route.

## How To Use In The Workflow

1. Finish Level 1-3 first. Do not use models to invent evidence.
2. Start Level 4 with the default Idea Platform route.
3. If needed, choose 1-2 additional models from this library to create contrast
   candidates.
4. Convert every model output back into the same Idea Platform record:
   cultural tension, brand truth, proof edge, audience shift, and six-check.
5. Recommend one route only when evidence and brand facts support it.

## Core Derivation Routes

| Route | Best When | How It Helps Level 4 |
|---|---|---|
| Default Idea Platform | The evidence reveals a cultural tension and the brand has credible proof | Converts tension, brand truth, and proof edge into a strategic platform |
| 奥美品牌大理想 | The work needs a belief-led platform with a strong cultural point of view | Tests whether the brand can stand for a bigger change than a product claim |
| 奥美品牌定位三角 | The work needs a clear positioning sentence | Sharpens audience, category, and reason-to-believe |
| 3C Strategy Triangle | The problem is strategic rather than only communicational | Checks fit across customer need, company capability, and competitor difference |
| 奥美蝴蝶方阵 The Butterfly | The user wants an end-to-end brand strategy-to-expression map | Links brand definition, central platform, and expression system |
| USP / SMP / ESP | The output needs one sharp proposition | Forces a single functional, emotional, or strategic claim |
| Challenger Brand | The brand is not the category leader and must break a convention | Finds a pointed stance against category habits |
| Cultural Branding | The category is shaped by social contradiction or identity pressure | Turns cultural conflict into a brand role |
| Golden Circle | The brand has a strong founding belief or purpose | Uses why/how/what to clarify the belief behind the platform |
| Brand Key / Essence | The brand needs a compressed identity core | Turns audience, benefits, values, proof, and distinction into an essence |
| Jobs To Be Done | The evidence is behavior-heavy and need-driven | Links functional, emotional, and social jobs to a usable platform |
| Get / To / By / So | The team needs an action-oriented bridge to execution | Expresses the desired audience shift and the brand action that creates it |

## Deep Notes For Common Routes

### Default Idea Platform

- Logic: cultural tension x brand truth x proof edge.
- Use when the input package includes evidence, brand facts, and competitor
  context.
- Output form: "Because [audience] is caught between [what is] and [what should
  be], [brand] should stand for [strategic belief] and prove it through [proof
  edge]."
- Strength: keeps the strategy grounded and avoids slogan-first thinking.

### 奥美品牌大理想

- Logic: cultural tension x the brand at its most credible and meaningful.
- Use when the desired output should feel belief-led rather than product-led.
- Translate its result into the Idea Platform record instead of leaving it as a
  separate slogan.
- Watchout: do not force a grand belief if the brand facts are thin.

### 奥美品牌定位三角

- Logic: target audience x category frame x reason-to-believe.
- Use when the work needs sharper positioning before or after Idea Platform
  generation.
- Helpful output: "For [audience], [brand] is the [category] that [distinctive
  reason]."
- Watchout: a positioning line can support the platform, but it is not always
  the platform itself.

### 3C Strategy Triangle

- Logic: customer need x company capability x competitor difference.
- Use when Level 4 feels too communications-led and needs a business strategy
  check.
- Map it back to the default system:
  - customer need -> cultural tension / audience pressure
  - company capability -> brand truth
  - competitor difference -> proof edge

### 奥美蝴蝶方阵 The Butterfly

- Logic: brand definition -> central idea -> brand expression.
- Use when the user asks how insight becomes expression across touchpoints.
- Map it to this skill:
  - left wing -> Level 1-4 evidence and strategy work
  - center -> Idea Platform
  - right wing -> Concept, Message House, and execution planning

### USP / SMP / ESP

- Logic: reduce the strategic claim to one clear proposition.
- Use when the Idea Platform feels too broad or too abstract.
- Output types:
  - USP: functional reason to choose
  - ESP: emotional reason to care
  - SMP: single-minded proposition for creative work

### Challenger Brand

- Logic: identify the category convention, choose what to sacrifice, and take a
  credible opposing stance.
- Use when the brand needs distinction against larger or more familiar players.
- Watchout: the challenger stance still needs product, behavior, or channel
  proof.

### Cultural Branding

- Logic: identify a cultural contradiction and give the brand a role inside it.
- Use when the evidence shows identity, social pressure, or values conflict.
- Watchout: this route needs strong cultural evidence; do not elevate a small
  pain point into a culture claim.

### Get / To / By / So

- Logic: get [audience] to [new belief/action] by [brand action], so [result].
- Use after an Idea Platform is chosen to make the strategic shift actionable.
- Good bridge into Concept and Message House.

## Model Selection Guide

- Cultural contradiction is strong: Default Idea Platform, 奥美品牌大理想,
  Cultural Branding.
- Positioning is unclear: 奥美品牌定位三角, 3C, Brand Key.
- Claim is too broad: USP / SMP / ESP, Get / To / By / So.
- Brand is smaller or needs edge: Challenger Brand, Blue Ocean / value
  innovation.
- Brand has a strong founding belief: Golden Circle, Brand Purpose.
- Work is moving into execution: Message House, Get / To / By / So, PESO,
  content model.

## Output Rule

No matter which route is used, normalize the result into
`assets/templates/idea-platform-record-template.md`. The model is a route; the
Idea Platform record is the shared output.
