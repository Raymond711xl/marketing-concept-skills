# Concept Strategy Controller 使用说明

Status: Draft / Alpha
Version: 0.1.0
License: No license yet. This project is not ready for reuse or redistribution.

这是一套面向中文品牌与中文网络环境的品牌策略工具组合。它帮助你从一个模糊 brief 开始，一步步走到可判断、可讨论、可修改的 Big Idea 和 Concept。

核心路径是：

```text
Brief -> 证据采集 -> 证据摘要 -> 洞察策略 -> Big Idea -> Concept
```

## 这套工具能做什么

- 帮你把一句模糊需求整理成项目启动包。
- 判断现在应该先调研、先清洗资料、先做洞察，还是直接进入策略判断。
- 在调研时保留来源、链接、日期、原话、置信度和限制。
- 把大量资料压缩成可读的证据模式，而不是直接跳到结论。
- 从证据中推导事实层、动机层、文化张力和品牌战略选择。
- 最终收束到 Big Idea 和 Concept。
- 让你随时停在某个阶段，单独讨论、修改或回退。

## 包含哪些 Skill

- `concept-strategy-controller`：总控。负责读 brief、判断路线、压缩阶段输出、维护完整资料池。
- `web-evidence-collector`：证据采集。负责公开网络资料、平台线索、素材、报道、社媒线索和 evidence pool。
- `evidence-summary-analysis`：证据摘要。负责清洗资料、分类、归纳证据模式，不做最终策略判断。
- `insight-strategy`：洞察策略。负责从证据进入 human truth、cultural tension、Brand Big IdeaL 和战略建议。

## 默认语言和市场规则

- 默认用中文工作和输出。
- 中文品牌、中文品类、中国市场项目，优先使用中文搜索词、中文平台和中文网络语境。
- 如果 brief 明确指定海外市场、英文交付、国际竞品、全球平台或跨境语境，再切换到英文或海外调研路径。
- 证据原文、用户原话、slogan、帖子标题、消费者评论尽量保留原语言。
- 跨 skill 的结构字段可以保留英文，例如 `Evidence Pool`、`Confidence`、`Raw quote`、`Insight Map`、`Big Idea Record`，这样方便后续迁移和交接。

## 如何开始一个项目

你可以直接给总控一个 brief，例如：

```text
我想为一个新中式茶饮品牌做品牌策略，目标是让它在年轻白领中建立“更轻松的日常奖励”心智。现在没有资料，希望先从竞品和社媒调研开始。
```

总控会先形成一个轻量项目启动包：

```markdown
- 用户原始 brief:
- 品牌 / 产品 / 品类:
- 市场 / 地域 / 语言:
- 商业问题:
- 传播目标:
- 目标人群:
- 已有资料:
- 证据状态:
- 目标终点:
- 推荐路径:
- 缺失输入:
```

这个启动包会跟随后续阶段，避免下层 skill 只收到压缩摘要而丢失你的原始意图。

## 如何深入某个环节

你可以随时输入“阶段指令”暂停主流程，单独深入某个阶段。建议使用下面这种明确文本，不使用 `@`，避免和 Codex 或其他工具的 mention 机制混淆。

- `进入：证据采集`：调整调研范围、平台、深度、证据缺口。
- `进入：证据摘要`：讨论分类、材料模式、来源偏斜、弱信号。
- `进入：策略洞察`：讨论 human truth、cultural tension、Brand's Best Self、战略选择。
- `进入：Big Idea`：展开、比较、压力测试或重写 Big Idea。
- `进入：Concept`：展开、比较、压力测试或重写 Concept。
- `查看：资料池`：查看、重组、审计完整后台资料。
- `回退到：证据摘要`：退回某个阶段重新处理。
- `继续`：恢复主流程。

当你这样做时，总控会暂停继续推进，把用户原话、当前判断和相关资料池一起带入该环节。

## 输出分两层

每个阶段都分成两层：

- `前台输出`：对话中直接给你看的短结果，用来判断下一步。
- `后台资料池`：总控维护的结构化项目记忆，保留证据、摘要、洞察图、策略候选、Big Idea 和 Concept 记录。

总控压缩的是阅读负担，不压缩证据、判断依据、矛盾、置信度和开放问题。

后台资料池默认不一定生成文件。它有三种呈现方式：

1. 默认在对话中只展示目录、关键变化和当前结论，避免刷屏。
2. 当你输入 `查看：资料池` 时，总控可以在对话中展开完整资料池或某个部分。
3. 当你要求导出时，总控再生成 Markdown 文件，例如 `backstage-dossier-[项目名].md`，用于保存完整项目记录。

## 系统边界

这套系统目前结束于 Concept。Concept 之后的文案、视觉、KV、脚本、提案、执行物料暂时不属于当前系统。后续可以继续扩展新的下层 skill。
